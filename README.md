# TokenFoundry

TokenFoundry is a library for creating and managing design tokens in a design system. It is built on top of the Design Tokens specification.

## Vision

The vision of TokenFoundry is to provide simple, flexible, and powerful tools for creating and managing design tokens. We want to make it easy for designers and developers to create and maintain design systems that are consistent, scalable, and easy to use.

In most cases, designers and developers speak different languages. Designers use tools like Figma, Sketch, and Adobe XD to create designs, while developers use code to implement those designs. TokenFoundry aims to bridge this gap by providing a common language for both designers and developers to communicate about design tokens.

TokenFoundry is designed to the Design Tokens Community Group specification, which is a standard for design tokens. By adhering to this specification, TokenFoundry ensures that design tokens created with the library are compatible with a wide range of tools and platforms.

Below are a list of desired products / features that we want to build with TokenFoundry:

- A design token editor that allows designers to create and manage design tokens in a visual interface.
  - Graphing support to visualize the relationships between tokens.
  - Tree view to organize tokens in a hierarchical structure.
- A CLI tool that allows developers to export design tokens to various formats (e.g., JSON, CSS, SCSS, etc.) and integrate them into their projects.
- A plugin for design tools (e.g., Figma, Sketch, Adobe XD) that allows designers to sync design tokens from their design files to the TokenFoundry library.
- A web-based platform for sharing and collaborating on design tokens with team members and stakeholders.
- A way to integrate with version control systems (e.g., Git) to track changes to design tokens over time and collaborate with other team members.
- A way to sync design tokens with design systems and component libraries to ensure consistency across all platforms and tools.
